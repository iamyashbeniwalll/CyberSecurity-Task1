# 🛡️ Cyber Security Internship - Task 1 Report
   
## ✅ Task Title:
**Scan Your Local Network for Open Ports**

---
   
## 🎯 Objective:
To learn how to discover open ports on devices in a local network using Nmap to understand network exposure.
        
---
        
## 🛠️ Tools Used:
- **Nmap** (Network Mapper)
- **Wireshark** (optional, not used in this task)
        
---

## 📝 Task Steps:
     
1. **Installed Nmap** on my MacBook using Homebrew:
   `bash'
   brew install nmap
      
2. Identified my local IP address and subnet:
   'bash'
   ipconfig getifaddr en0
  ( My IP: 192.168.1.x → Local network: 192.168.1.0/24 )
        
3. Scanned the network for live hosts and open ports:
   'bash'
    nmap -sT -Pn 192.168.1.0/24

4.Targeted a specific IP (example: 192.168.1.7):
   'bash'
   nmap -sT -Pn 192.168.1.7

5.  Result:
        •       Host was up.
        •       All scanned ports were closed (conn-refused).
        •       No open ports were detected on 192.168.1.7.


    Observations:
        •       Many devices on the local network refused connections on all scanned ports.
        •       This is a good security practice – unnecessary open ports can be exploited.
        
    Research on Ports & Services:
     Port                        Typical Service                      Risk if Open
      22                              SSH                         Unauthorized access
      80                              HTTP                        Unencrypted traffic
      443                             HTTPS                       Generally secure
      3389                            RDP                         Remote desktop exploits

    Potential Security Risks of Open Ports:
        •       Open ports can expose services to unauthorized users.
        •       Can be used as an entry point for malware or exploitation.
        •       Information leakage or DoS vulnerabilities.
